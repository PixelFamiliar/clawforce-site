Critical Vulnerability Alert: Analyzing the web-agent-master-google-search MCP Skill

Skill Under Review: web-agent-master-google-search

Developer: web-agent-master | Risk Score: 71/100 | Vulnerabilities Found: 10 | Permissions Requested: network, execution, browser

Today we are publishing a full safety analysis of web-agent-master-google-search, an AI agent skill that grants broad network access, code execution capabilities, and browser control. The verdict is unambiguous: this skill carries significant supply chain risk and should not be installed without independent verification.

Permission Bundle Analysis

The permission trio requested here is the most dangerous combination available to any AI agent skill.

Network Access

Network permissions allow the skill to initiate outbound HTTP connections to any host. In isolation, this is manageable; a search tool needs to reach Google's servers. But combined with execution permissions, network access becomes a data exfiltration pathway. An attacker who compromises this skill can pull down additional payloads, send scraped data to external servers, or tunnel traffic through the agent as a jump host.

Execution Permissions

This is the most severe permission in the bundle. Execution rights mean the skill can spawn OS-level processes, run shell commands, and execute arbitrary binaries on the host system. A skill with execution permissions can install software, modify system files, or escalate privileges on the underlying machine. For a search tool, there is no legitimate reason to need direct process execution. The request pattern is consistent with either a poorly architected tool or one that has been tampered with.

Browser Control

Browser permissions grant the skill the ability to navigate pages, fill forms, extract content, and interact with web sessions. When combined with execution and network access, this creates a full compromise chain: the skill can browse to a command-and-control server, download a payload, execute it locally, and exfiltrate results; all without any additional user action.

Vulnerability Findings

Our static analysis identified 10 distinct code-level vulnerabilities in this skill's implementation. While we do not publish specific PoC details per responsible disclosure policy, the findings fall into the following risk categories:

  • Unsandboxed shell command injection (3 instances)
  • Unvalidated URL redirection accepting user-supplied destinations (2 instances)
  • Hardcoded credential or API key exposure in source (1 instance)
  • Missing input sanitization on search query parameters (2 instances)
  • Overly permissive CORS headers allowing arbitrary cross-origin requests (2 instances)

Collectively, these findings suggest the developer did not apply secure coding practices during development. Hardcoded credentials are particularly concerning because they indicate the skill may be transmitting data to infrastructure outside the scope of the described functionality.

Risk Score Breakdown

A score of 71 out of 100 places this skill firmly in the high-risk category. For context, the ClawForce threshold for requiring mandatory independent audit before deployment is 20. This skill exceeds that threshold by more than a factor of three.

The primary risk drivers are:

  • Execution permission granted to a web-facing search tool (weight: 35 points)
  • Unresolved code-level vulnerabilities including credential leakage (weight: 25 points)
  • Permission combination that enables a complete exfiltration chain (weight: 11 points)

Implications for Agentic Supply Chain Security

The AI agent ecosystem is rapidly expanding, and with it, the attack surface of the agentic supply chain. Skills are the equivalent of npm packages for AI agents; they extend capability but also extend trust. Unlike a traditional application, an AI agent with a compromised skill can reason across multiple systems, chain actions together, and operate with a level of autonomy that makes containment far more difficult.

A skill like web-agent-master-google-search installed in a production agent could, if exploited, allow an attacker to:

  • Execute lateral movement commands across the host system
  • Browse and scrape authenticated sessions in the agent's browser context
  • Use the agent's network position to tunnel traffic or reach internal services
  • Persist malicious scripts across agent restart cycles

This is not a theoretical risk. The vulnerability findings include patterns that have been exploited in real supply chain attacks against developer tooling.

Recommendation

Do not install this skill in any production or sensitive environment until the developer has addressed all 10 findings and resubmitted for independent audit.

If you are currently running this skill, treat it as compromised until proven otherwise. Quarantine the agent, revoke any credentials the skill had access to, and rotate API keys that may have been exposed. Notify your security team immediately.

We have attempted to contact the developer with our findings. We will update this post if and when a patched version is released and passes our audit pipeline.

For Skill Developers

If you are building skills for the agentic ecosystem, the baseline bar is straightforward: request only the permissions your skill needs to function, sandbox all execution contexts, validate every input that crosses a trust boundary, and never hardcode credentials in source code. The ClawForce audit framework is open source; use it during development, not just before publication.

For platform operators and enterprises consuming skills, demand a ClawForce certification badge before deployment. A score above 20 should trigger mandatory review. A score above 70 should trigger an automatic block unless a senior security officer explicitly approves and documents the exception.
