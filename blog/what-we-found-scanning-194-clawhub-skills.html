Permission Scope Distribution

We infer 8 permission scopes from skill descriptions. When a skill mentions "HTTP", "API call", or "fetch", we tag it with the network scope. When it mentions "execute", "shell", or "subprocess", we tag it with execution. Here's how often each scope appeared:

The pattern is clear: skills that combine execution + credentials + another scope consistently score highest. A skill that can both run code and access your API keys is, by definition, higher risk than one that can only search the web.

Important: A high risk score doesn't mean a skill is malicious. It means the skill has a broad access surface and warrants closer inspection before deployment. Many legitimate tools need these permissions to function.

What This Means for Agent Operators

If you're running an AI agent that installs skills from ClawHub, you should know what each skill can access. The current ecosystem has no mandatory permission declaration. Skills describe what they do in plain text, and operators trust that description. Our scanner infers what a skill likely needs based on that description.

The gap between what's described and what's actually required is where risk lives. A skill called "web search" shouldn't need execution or credential scopes, but some do. Those are worth investigating.

Methodology

ClawForce Scanner v2.3.0 performs metadata-level analysis. We do not have access to the source code of ClawHub skills (their API doesn't expose it). Our analysis is based on:

  • Skill descriptions and changelogs (pattern matching against 33 security signals)
  • Permission scope inference (8 scopes matched against description keywords)
  • Supply chain signal detection (runtime installs, curl|bash patterns)
  • ClawHub moderation verdict cross-referencing
  • Community adoption metrics (downloads, stars, version count)

This is a metadata scan, not a code audit. The risk scores reflect our best assessment of what a skill describes itself as doing. Real code-level analysis would require access to the deployed skill packages.
