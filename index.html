How the Scanner Works

Three stages. Fully deterministic. No black boxes.

Stage 1 — Static Analysis
skill_scanner.py
AST parsing • 22 regex patterns • Secret detection (AWS, Stripe, GitHub tokens) • Base64/hex obfuscation • SKILL.md structure validation • Dependency audit
Output: risk_score (0–100), findings[]
→
Stage 2 — Intent Analysis
intent_analyzer.py
SKILL.md claim extraction • Code behavior detection • Claim-vs-behavior comparison • Comment stripping (anti-prompt-injection) • Manifest validation
Output: mismatch_score (0–100), mismatches[]
→
Stage 3 — Sandbox Execution
certifier.py
Docker isolation • Network activity monitoring • Syscall tracing • Behavioral analysis • Human security review sign-off
Output: certification + badge + signed JSON

Security & Transparency

We disclose our threat model, our limits, and how we operate — because real trust is earned through specifics, not badges.

Open, Deterministic Scanner
22 pattern rules, AST analysis, and intent comparison — every check is visible and auditable. No black box, no opaque ML scoring.
Hash-Pinned Certificates
Every certification is tied to the exact SHA-256 hash of the skill bundle. Update one file and the badge invalidates — no silent swaps.
Reproducible Results
Certificates include scanner version, timestamp, and commit hash. Anyone can re-run the exact same scan and independently verify.
We Scan Ourselves First
Every skill we publish is certified through our own pipeline before release. Check the gallery — our skills are listed with full reports.
Revocation API
If a zero-day is discovered post-certification, we revoke instantly. Platforms can query our CRL endpoint to check real-time badge status.
Skill Manifest Standard
We require skills to declare permissions in a skill-manifest.json — network egress, file paths, shell access — before certification.

What We Defend Against

We're specific about what our certification catches — and what it doesn't.

What We Catch
Hardcoded secrets, obfuscated payloads, undeclared network egress, credential harvesting, base64 exfiltration, crypto miners, privilege escalation, and SKILL.md/code intent mismatches.
Known Limitations
Certification is a point-in-time assessment. We cannot guarantee future safety if upstream dependencies are later compromised, or if novel zero-day techniques emerge.
Security Reporting
Found a vulnerability in a certified skill? Email pixel@pixelfamiliar.ca with reproduction steps. We respond within 48 hours and coordinate responsible disclosure.

Developer API

Integrate ClawForce scans into your CI/CD pipeline.

curl
# Submit a skill for Tier 1 scan
curl -X POST https://api.clawforce.ca/v1/scan \
  -H "Content-Type: application/json" \
  -d '{"repo": "github.com/you/skill"}'

# Check scan status
curl https://api.clawforce.ca/v1/scan/SCAN_ID

# Verify a certificate
curl https://api.clawforce.ca/v1/verify/HASH
python
import requests

# Submit scan
r = requests.post(
  "https://api.clawforce.ca/v1/scan",
  json={"repo": "github.com/you/skill"}
)
scan_id = r.json()["id"]

# Get results
result = requests.get(
  f"https://api.clawforce.ca/v1/scan/{scan_id}"
).json()
