How to Verify AI Agent Skills

Every AI agent skill you add to your workflow is a potential attack vector. Here's a 5-step framework for vetting them like a security pro.

1. Hash Verification (The Baseline)

# Always verify skill bundle integrity
openssl dgst -sha256 skill-bundle.tar.gz
# Compare against publisher's published hash

Why it matters: Hash verification ensures the skill hasn't been tampered with in transit. But it only validates integrity, not identity. A valid hash from an attacker is still a valid hash.

2. Identity Chaining (The Missing Piece)

SHA-256 hashes alone don't tell you who published the skill, whether that entity is legitimate, or if an imposter pushed a "new version." The fix: cryptographic signatures that bind publisher identity, content hash, timestamp, and permission scope into a single verifiable chain.

3. Permission Scope Audit

Every skill should expose its capabilities explicitly. Review the manifest and ask: does this skill need everything it claims?

// manifest.json
{
  "name": "deploy-helper",
  "permissions": ["read:repo", "write:repo", "exec:npm"],
  "network": ["api.cloudservice.com"],
  "file_access": ["workspace/*"]
}

Red flags to watch for:

exec:*

Overly broad execution permissions

net:*

No network restrictions declared

fs:/**

Unbounded filesystem access

See the full Scopes Reference for all 13 scope types and their risk levels.

4. Prompt Injection Analysis

Review the skill's prompt templates for dynamic variable injection (user input into prompts without sanitization), role escalation (tricking the LLM into higher-privilege actions), and context pollution (overriding system instructions).

# Test for injection vulnerability
test_input = "'; rm -rf / #"
result = skill.execute(test_input)
# If result executes shell commands → vulnerable

For a deep dive on injection patterns, see our Prompt Injection Detection Guide.

5. Dependency Audit

Skills depend on npm packages, Python modules, Docker images, and external APIs. Each is a potential supply chain attack. Verify package integrity, maintainer reputation, and known CVEs. ClawForce generates a CycloneDX SBOM for every scanned skill to make this traceable.

Quick Verification Script

#!/bin/bash
# verify-skill.sh — The dev's verification workflow

SKILL_DIR=$1
PUBLISHER_KEY="clawforce:pub:abc123"

echo "=== Verifying skill: $SKILL_DIR ==="

# 1. Check manifest exists
[ -f "$SKILL_DIR/manifest.json" ] || { echo "❌ No manifest"; exit 1; }

# 2. Verify identity signature
clawforce verify --identity "$PUBLISHER_KEY" --skill "$SKILL_DIR"

# 3. Audit permissions
cat "$SKILL_DIR/manifest.json" | jq '.permissions'

# 4. Check dependencies
[ -f "$SKILL_DIR/package.json" ] && npm audit --prefix "$SKILL_DIR"

echo "✅ Verification complete"
